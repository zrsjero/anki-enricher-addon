# Anki English Enricher Add-on

Anki English Enricher is a Python add-on that enriches vocabulary notes by filling missing data in configured fields.

The add-on workflow:
- read notes of one configured note type;
- fill missing IPA, definition, examples, and audio;
- save generated media files to Anki `collection.media`;
- show a summary with counters and failed words/fields.

## Current Status

As of **March 18, 2026**, the add-on has a working end-to-end pipeline with:
- IPA from Free Dictionary API;
- definition and examples from selected text provider (currently `ollama`);
- audio generated via native macOS `say` and stored as `.aiff`.

## Features

- Adds menu action: `Tools -> English Note Enricher`
- Shows one run dialog with:
  - deck selection (`[All decks]` + all deck names)
  - default deck selection: current/last active deck (if available), otherwise `[All decks]`
  - provider selection for `Definition` and `Example` (provider registry)
- Targets one note type from config (`note_type_name`)
- Verifies required fields exist in the model before processing
- Updates only empty target fields
- Optional lowercasing of first character in source fields (`English`, `Russian`)
- Normalizes generated `Definition`/`Example` first letter to uppercase
- Cleans malformed example payloads (e.g., JSON fragments mixed into sentence text)
- Generates deterministic audio filenames
- Writes audio files into Anki media folder
- Produces run summary in a popup dialog, including failed words and failed fields

## Enrichment Rules

Expected fields:
- `English`
- `Russian`
- `IPA`
- `Definition`
- `Example`
- `EnglishAudio`

Target fields (filled only when empty):
- `IPA`
- `Definition`
- `Example`
- `EnglishAudio`

### IPA

- Source: Free Dictionary API (`https://api.dictionaryapi.dev/api/v2/entries/en/{word}`)
- Selection strategy:
  1. `entry["phonetic"]`
  2. first valid `phonetics[].text`
- Normalization: slash-wrapped format like `/wɜːd/`
- IPA is dictionary-only

### Definition

- Source: selected text provider (`text_provider`), currently `ollama`
- Generated as one short definition sentence
- Applies quality checks
- Applies text cleanup + first-letter uppercase normalization

### Examples

- Source: selected text provider (`text_provider`), currently `ollama`
- Generates candidate examples with quality filtering
- Removes malformed structured fragments (JSON-like tails/lines)
- De-duplicates case-insensitively
- Limits count by config (`example_count`)
- Stores output as HTML line breaks (`<br>`)
- Applies first-letter uppercase normalization

### Audio

- Backend: `macos_say`
- Uses native `say -o <temp>.aiff <text>`
- Reads generated bytes and removes temp file
- Saves file to media folder using deterministic filename
- Stores Anki tag format: `[sound:<filename>]`

## Configuration

Config is stored in `config.json`.

Current repository config:

```json
{
  "note_type_name": "JeEng",
  "fields": {
    "english": "English",
    "russian": "Russian",
    "ipa": "IPA",
    "definition": "Definition",
    "example": "Example",
    "english_audio": "EnglishAudio"
  },
  "example_count": 2,
  "text_provider": "ollama",
  "text_provider_max_attempts_per_word": 2,
  "normalize_source_first_char_lowercase": true,
  "ollama": {
    "enabled": true,
    "base_url": "http://127.0.0.1:11434",
    "model": "qwen2.5:3b-instruct",
    "timeout_seconds": 25,
    "temperature": 0.6
  },
  "audio_prefix": "jeeng",
  "audio_backend": "macos_say"
}
```

### Config Keys

- `note_type_name`: target note type name
- `fields`: mapping from logical keys to actual note field names
- `example_count`: max number of examples to store in `Example`
- `text_provider`: active provider key for `Definition` and `Example`
- `text_provider_max_attempts_per_word`: shared retry count for `Definition`/`Example` generation across all text providers
- `normalize_source_first_char_lowercase`: lowercases first char in `English`/`Russian`
- `ollama.*`: local Ollama settings used when `text_provider=ollama`
- `audio_prefix`: prefix in generated audio filenames
- `audio_backend`: audio backend (`macos_say`)

## Provider Extension Pattern

Definition/example generation now uses a provider registry:
- file: `providers/text_provider_registry.py`
- each provider supplies two callables:
  - `generate_definition(word)`
  - `generate_examples(word, count)`

To add a new LLM provider, the main work is:
1. add new provider module under `providers/`
2. register it in `providers/text_provider_registry.py`
3. set `text_provider` in config (or select it in UI)

Core service logic does not need dictionary-vs-ollama branching anymore.

## Installation

1. Open Anki add-ons folder for your profile.
2. Place this project under `addons21/anki_enricher_addon`.
3. Restart Anki.
4. In Anki, run `Tools -> English Note Enricher`.

## Usage

1. Ensure target note type exists and fields match config.
2. Keep `English` populated for notes you want to enrich.
3. Leave target fields empty if you want auto-fill.
4. Run `Tools -> English Note Enricher` and choose:
   - deck (default is current/last active deck)
   - provider for definition/examples (currently `Ollama (local)`)
5. Review summary popup.

Summary example:

```text
Processed: 8
Updated: 0
Skipped: 0
IPA updated: 0
Definition updated: 0
Example updated: 0
Audio updated: 0
Errors: 8

Failed words:
1) hesitantly [ipa]
2) umbilic [ipa, audio]
3) fraudster [example]

```

## Logging

Field-level failures are logged with `logger.warning` in `services/enrich_service.py` with:
- note id
- word
- field
- failure reason

To view logs on macOS:
- `/Applications/Anki.app/Contents/MacOS/launcher`

## Troubleshooting

### "Missing required fields"

- Field names in config do not match note model fields.

### No IPA found

- Dictionary API has no phonetic data for this word.
- API/network issue.

### No definition/examples found

- Text provider unavailable/misconfigured.
- For Ollama: runtime not running, model missing, or output failed quality checks.

### Audio not generated

- `say` command unavailable (non-macOS or PATH issue).
- Empty/invalid `English` text after normalization.

## Known Limitations

- Text provider registry currently has one production provider: `ollama`
- Audio backend currently has one production option: `macos_say`
- Audio format is `.aiff` (larger than compressed formats)
- No config schema validation layer yet
- No automated tests yet
